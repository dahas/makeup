What is it?

makeUp is an easy to use PHP framework and a compilation of popular frontend libraries. It is a package that includes almost everything you need to build awesome web applications from the ground up.

Read more ...

Requirements

In order to get makeUp to work, you need PHP 5.6 or above, a MySQL database and a local webserver. If you have installed Xampp lately (or something similar) you should already be set up.

Read more ...

Download

Either download makeUp from GitHub, or make a clone: 

$ git clone https://github.com/dahas/makeup.git

Get started

  1. Create folder "makeup", that is within the root directory of your project. 
    $ mkdir makeup
  2. Install dependent libraries 
    $ cd makeup
$ php composer.phar install
  3. Create file index.php in your root folder. Paste the following lines into it: 
    %CODE%
  4. Launch your app in the web browser.

Create a module

  1. Use the makeup.phar CLI to create a module with a specific name. 
    $ cd makeup
$ php makeup.phar --create-module YourModule
  2. Check the info in the terminal. All files and folders must have been successfully created.
  3. Launch your app in a browser and add ?mod=your_module to the URL.

Use a module

  • Make an instance of your module and render it. 
    $yourModule = Module::create("your_module");
$yourModule->render();
  • Instead of render() you can call any other task you have defined in your module.

Markers

  • Markers are special strings, that you place in the HTML template. 
    <span>##THIS_IS_A_MARKER##</span>
  • In the php controller you use an array to assign the marker. The $marker array must then be injected into the parse method of the template object. 
    public function build()
{
    $marker = [];
    $marker["##THIS_IS_A_MARKER##"] = "makeUp is great!";
    return $this->getTemplate()->parse($marker);
}
  • This is how the marker looks after the template has been rendered. 
    <span>makeUp is great!</span>

Subparts

  • Markers can also be replaced with code segments. Those segments are called subparts. Subparts are a great way to render large lists or tables. In this example the marker ##TMPL_ROWS## will be replaced dynamically with multiple rows. 
    <table class="table table-striped">
    <thead>
        <tr>
            <th>#</th>
            <th>First Name</th>
            <th>Last Name</th>
            <th>Username</th>
            <th>Country</th>
        </tr>
    </thead>
    <tbody>
        ##TMPL_ROWS##
    </tbody>
</table>
  • Create a special template file "rows.html" in the view folder with the following content. As you can see, the template contains markers, that are treated with ease, as we already learned above. 
    <tr>
    <td>##UID##</td>
    <td>##NAME##</td>
    <td>##AGE##</td>
    <td>##CITY##</td>
    <td>##COUNTRY##</td>
</tr>
  • Instantiate a template object for "rows.html" in the build method of the controller. Use the parse function of that object within the foreach loop and add rendered content to the main marker. 
    public function build()
{
    $marker = [];
    $rowsTmpl = $this->getTemplate("rows.html");
    $marker["##TMPL_ROWS##"] = "";

    $items = [
        ["uid" => 1, "name" => "Jaques", "age" => 27, "city" => "Paris", "country" => "France"],
        ["uid" => 2, "name" => "Karin", "age" => 31, "city" => "Berlin", "country" => "Germany"],
        ["uid" => 3, "name" => "Paul", "age" => 54, "city" => "London", "country" => "Great Britain"],
        ["uid" => 4, "name" => "Vittoria", "age" => 40, "city" => "Rome", "country" => "Italy"],
    ];

    foreach ($items as $item) {
        $mkArr = [
            "##UID##" => $item["uid"],
            "##NAME##" => $item["name"],
            "##AGE##" => $item["age"],
            "##CITY##" => $item["city"],
            "##COUNTRY##" => $item["country"]
        ];
        $marker["##TMPL_ROWS##"] .= $rowsTmpl->parse($mkArr);
    }

    return $this->getTemplate()->parse($marker);
}
